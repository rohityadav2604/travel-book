## 1. Executive Summary

MemoryBook is a web application that transforms a user's travel photo collection into a beautifully designed, print-ready vintage photo book. Users upload 30–50 photos, the platform analyses each image for composition, colour palette, subject matter, and emotional tone, then intelligently places them into aesthetically curated book templates rooted in a vintage-retro design language. The final output renders as an interactive, page-turning preview inside the browser — and is downloadable as a high-resolution PDF.

The product is built on two foundational pillars: **intelligent image curation** (selecting and sequencing the best photos) and **expressive book design** (placing those photos in layouts that feel warm, nostalgic, and deeply personal). Everything else — onboarding, export, sharing — is secondary to making those two things magical.

> **Core Promise:** Every book generated by MemoryBook should make the user pause, smile, and feel something — the same feeling as flipping through a well-loved photo album from the past.

## 2. Problem & Opportunity

### 2.1 The Problem

Travel photos accumulate by the hundreds. Most never leave a phone camera roll. The rare user who wants to preserve memories faces a choice between generic drag-and-drop book builders that produce lifeless grids, or expensive professional services that take weeks. Neither option captures the emotional warmth people associate with handmade travel journals and old photo albums.

### 2.2 The Opportunity

AI image analysis combined with thoughtful, template-driven design can automate the hardest part — curation and placement — while preserving the artistic soul that makes a photo book feel personal. A vintage-retro visual identity reinforces the nostalgia that users are already chasing when they decide to preserve their memories.

### 2.3 Target Users

| Persona | Description | Key Need |
|---|---|---|
| **The Solo Traveller** | 25–40, documents long solo trips with 300+ photos | Reduce overwhelming choice; produce something shareable |
| **The Couple / Family** | 28–45, wants a keepsake from a holiday | Easy, beautiful result with minimal effort |
| **The Creative Documenter** | Any age, treats travel as storytelling | Artistic templates that match their aesthetic vision |
| **The Gift Giver** | Wants to print a book for a partner or parent | High-quality, downloadable PDF ready for print |

## 5. Feature 1 — Smart Photo Upload & Curation

> **Feature Goal:** Accept a raw dump of 30–50 travel photos and return a curated, ranked, and emotionally coherent subset perfectly sized for the chosen book template — with zero manual effort from the user.

### 5.1 Upload Experience

The upload surface is the first impression. It must feel intentional and premium, not like a file picker.

#### 5.1.1 Constraints

- Minimum 30 photos, maximum 50 photos per book session.
- Accepted formats: JPEG, HEIC, PNG, WebP.
- Maximum file size per photo: 25 MB. Maximum total session upload: 500 MB.
- Photos must be uploaded in a single batch (no incremental adds in V1).

#### 5.1.2 Upload UX Flow

- Drag-and-drop landing zone occupies the majority of the viewport on the upload screen.
  - Vintage-styled dashed border, sepia-toned icon, and handwritten-style instructional copy.
- File picker fallback for non-drag environments (mobile, older browsers).
- Chunked upload with resumable upload support (tus protocol or equivalent) for large batches.
- Per-photo upload progress displayed as a horizontal film-strip loading animation.
- If < 30 or > 50 photos are selected, an inline validation message blocks progression with a clear, friendly explanation.

### 5.2 Image Analysis Pipeline

After upload completes, each image passes through an analysis pipeline before curation. This pipeline runs server-side and should complete within 60 seconds for 50 images (targeting ~1.2 s / image on average).

#### 5.2.1 Per-Image Analysis Signals

| Signal | Method | Output |
|---|---|---|
| **Technical quality** | Blur detection (Laplacian variance), exposure histogram | Quality score 0–1 |
| **Composition** | Rule-of-thirds saliency, subject centring, horizon detection | Composition score 0–1 |
| **Colour palette** | Dominant hue extraction, warmth/coolness, saturation | Palette tags + warmth score |
| **Subject detection** | On-device vision model: people, landmarks, nature, food, architecture | Subject label array |
| **Emotional tone** | Combination of lighting, subject, composition heuristics | Tone: joyful / serene / adventurous / intimate |
| **Near-duplicate detection** | Perceptual hash (pHash) with Hamming distance ≤ 8 | Duplicate cluster ID |
| **Orientation & crop** | EXIF data + subject bounding box | Canonical crop rect |

#### 5.2.2 Curation Algorithm

After per-image scoring, the algorithm applies the following logic to produce a final curated set:

- Discard all images with quality score < 0.35 (blurry, grossly over/under-exposed).
- From each near-duplicate cluster, retain only the highest-scoring image.
- From the surviving set, select up to N images that maximise subject variety and emotional arc diversity, where N is the target image count for the chosen book template (typically 28–44 slots).
- Sequence the selected images: opening spread uses the highest-scoring "joyful" or "adventurous" hero shot; closing spread uses a "serene" or "intimate" image; body follows a loose narrative arc built from subject labels and geo metadata when available.

### 5.3 Curation Review UI

Before entering the book renderer, users see a curation summary screen showing which photos were selected, which were filtered out and why, and allowing lightweight adjustments.

#### 5.3.1 Selected Photos Panel

- Film-strip display of curated photos in proposed sequence order.
- Drag-to-reorder within the strip (sequence hint — actual layout decisions belong to the template engine).
- Tap/click on a photo to mark it as "must include" (pinned) or "exclude".

#### 5.3.2 Filtered Photos Panel

- Collapsed by default. Expandable to reveal photos that were removed.
- Each removed photo shows its rejection reason: "Too blurry", "Duplicate of another shot", "Over-exposed".
- Users can override and reinstate any filtered photo; doing so triggers a re-run of the placement step only (not full re-analysis).

#### 5.3.3 Proceed Gate

- "Create My Book" CTA is only active when the curated set contains between the template's minimum and maximum image count.
- If the user's overrides push the count out of range, the CTA is disabled and the delta is shown ("Add 2 more photos or remove 1").

## 6. Feature 2 — Book Template Engine & Web Renderer

> **Feature Goal:** Compose a complete, aesthetically beautiful vintage photo book by intelligently placing curated images into expressive template layouts, then render it as a fully interactive page-turning experience in the browser before final PDF export.

### 6.1 Design Language & Aesthetics

The entire product visual identity is grounded in a vintage-retro design system. Every template must feel as though it could have been produced by a talented designer in the 1960s–1980s using analogue tools — warm, tactile, imperfect in the most deliberate way.

#### 6.1.1 Core Design Tokens

| Token Category | Values & Rationale |
|---|---|
| **Colour Palette** | Warm ambers (`#C8860A`), cream ivory (`#FAF3E0`), burnt sienna (`#A0522D`), aged sage (`#7D9B76`), ink black (`#2B2318`). No pure whites or cool blues. |
| **Typography** | Serif headline face (Playfair Display or similar) for chapter titles and pull quotes. Handwritten script face (Caveat or similar) for captions. Small monospaced for dates/coordinates. |
| **Textures** | Subtle paper grain overlay (PNG sprite, multiply blend mode) on page backgrounds. Light vignette on full-bleed images. Optional torn-edge treatment on select photo crops. |
| **Photo Treatment** | Slight warm colour grade (+10 temperature, +5 vibrance) applied at render time. Optional sepia overlay on hero images (user toggle). Polaroid-style white border on small accent photos. |
| **Spatial Rhythm** | Generous whitespace. Asymmetric layouts preferred over centred grids. Overlapping photo crops for a scrapbook feel on select spreads. |
| **Iconography** | Hand-drawn-style line icons (compass rose, postage stamp outline, camera silhouette) used as decorative accents only. |

### 6.2 Book Structure

| Book Element | Description | Page Count |
|---|---|---|
| **Cover** | Full-bleed hero image, title (auto-generated or user-edited), subtitle, decorative stamp | 1 page |
| **Inside Front** | Handwritten-style dedication quote + small accent photo | 1 page |
| **Chapter Divider** | Full-bleed atmospheric image with chapter label; auto-inserted every 6–8 photos | 1 per chapter |
| **Standard Spread** | Two-page layout; the core repeating unit of the book | Variable |
| **Inside Back** | Map illustration (decorative, not functional) + trip stats | 1 page |
| **Back Cover** | Solid textured background, minimal branding, ISBN barcode placeholder | 1 page |

### 6.3 Spread Layout Templates

The template engine operates at the level of a two-page spread (a left page and a right page rendered as one unit). V1.0 ships with five spread templates. Each template defines a set of photo slots with precise placement rules.

#### 6.3.1 Template: "Grand Vista"

A single hero image spanning the full two-page bleed. Used for the most compositionally strong landscape shots. One slot, one image. A caption area sits in the lower-left gutter in handwritten script.

#### 6.3.2 Template: "The Journal Page"

Asymmetric layout: left page is a large portrait or square photo with a torn-edge bottom treatment; right page has two small landscape photos stacked vertically with generous cream background, a pull-quote text block, and a date annotation in monospace. Three photo slots total.

#### 6.3.3 Template: "Polaroid Wall"

Five to six photos of varying sizes scattered with slight random rotations (±3°) on a textured cream background, imitating photos pinned to a corkboard. A hand-drawn pin icon appears above the central image. Five photo slots.

#### 6.3.4 Template: "Golden Hour"

Full-bleed warm-toned landscape on the left page. Right page: a cream column with three portrait photos at reduced scale, separated by thin amber rules, each with a coordinate caption. Four photo slots total.

#### 6.3.5 Template: "The Contact Sheet"

A grid of eight equal-size photos on a dark ink background, imitating a photographer's contact sheet. Tight 4px gutters. A strip of film-frame decorations runs along the top and bottom edges. Eight photo slots. Used for high-density sections.

### 6.4 Image-to-Slot Placement Rules

The placement algorithm must match each curated photo to the most appropriate slot across all templates in the book. This is the most critical quality driver in the product.

#### 6.4.1 Slot Affinity Scoring

Each template slot has a declared affinity profile. The placement engine scores every candidate photo against every open slot and assigns photos to maximise total affinity across the whole book.

| Slot Property | How Photos Are Matched |
|---|---|
| **Aspect ratio preference** | Hero slots prefer 16:9 or panoramic; portrait slots prefer 3:4 or 2:3; square slots prefer 1:1. Score penalises heavy cropping required. |
| **Subject preference** | Grand Vista and Golden Hour prefer landscapes/architecture. Polaroid Wall prefers people and candid moments. Contact Sheet is subject-agnostic. |
| **Tonal preference** | Opening spreads prefer "joyful" / "adventurous" tone. Mid-book spreads mix tones. Closing spreads prefer "serene" / "intimate". |
| **Colour harmony** | Adjacent slots within a spread should not have clashing dominant hues. Warm-dominant photos are preferred for amber-background templates. |
| **Scale sensitivity** | Small accent slots (< 30% of spread area) penalise photos with many small details; they favour bold, high-contrast subjects. |

#### 6.4.2 Assignment Algorithm

- Model placement as a linear assignment problem: photos are workers, slots are tasks, affinity score is the cost matrix.
- Solve with the Hungarian algorithm (O(n³)) — tractable for ≤ 50 photos and ≤ 60 slots.
- Apply hard constraints first (aspect ratio crop would exceed 30% of image area → invalid assignment).
- Re-run assignment if user pins or excludes photos from the curation review screen.

### 6.5 Web Renderer

The rendered book is the centrepiece of the product. It must be beautiful, performant, and interactive.

#### 6.5.1 Rendering Architecture

- Each spread is rendered as an HTML/CSS layer with absolute-positioned image elements and SVG decorative overlays.
- Photos are served from a CDN-backed object store at the display resolution required for the rendered viewport.
- Paper texture and vignette effects are applied via CSS `mix-blend-mode` and pseudo-element overlays — no canvas-based compositing in the browser.
- A high-fidelity version of each spread is rendered server-side (headless Chromium + Puppeteer) for PDF export at 300 DPI. The browser renderer is a 96 DPI approximation.

#### 6.5.2 Page-Turn Interaction

- Horizontal swipe and arrow-key navigation between spreads.
- CSS 3D perspective transform simulates a page-turn animation (no JavaScript animation libraries — pure CSS transitions).
- A thumbnail strip at the bottom of the renderer allows direct navigation to any spread.
- Current spread index and total spread count displayed as a subtle page counter.

#### 6.5.3 Inline Editing (Light)

- Tap any caption area to enter edit mode; type a replacement caption. Changes are stored client-side and reflected in the PDF export.
- Tap the book title on the cover to rename it.
- No photo repositioning within slots in V1 (deferred — this is P1 for V1.1).

## 7. PDF Export & Download

### 7.1 Export Specifications

| Property | Value |
|---|---|
| **Output format** | PDF/X-3 (print-safe) and standard PDF (screen) |
| **Resolution** | 300 DPI (print), 150 DPI (screen/web) |
| **Colour space** | sRGB for screen PDF; CMYK conversion available as V1.1 enhancement |
| **Page size** | A4 portrait (210 × 297 mm) in V1.0 |
| **Bleed** | 3 mm bleed on all edges in print PDF; none in screen PDF |
| **Fonts** | All fonts embedded; subsetted |
| **File size target** | ≤ 60 MB for a 40-photo book at 300 DPI |

### 7.2 Export UX

- "Download Book" CTA is prominently placed in the renderer toolbar.
- User selects Print (300 DPI) or Screen (150 DPI) quality.
- Export is processed server-side; a progress indicator with estimated time is shown (target: ≤ 90 seconds for 300 DPI).
- Download is delivered via a signed, time-limited URL (expires in 24 hours). User is encouraged to save the file locally.

## 8. Technical Architecture

### 8.1 System Overview

MemoryBook is a cloud-native web application built entirely in TypeScript. The architecture uses a Next.js 15 monorepo as the core, with async Node.js workers behind a BullMQ job queue for compute-heavy operations (thumbnail generation, PDF rendering). There is no Python in v1.

> **Stack decision (v1):** All-TypeScript — Next.js 15 API Routes replace a separate API service; Node.js workers replace Python services. Image analysis is deferred to v2; v1 ships with manual photo reordering and simple aspect-ratio-based placement.

### 8.2 Component Map

| Component | Technology / Notes |
|---|---|
| **Frontend SPA** | Next.js 15 (App Router). React Server Components for renderer layout. Client components for interactive page-turn and upload. All source in `src/`. |
| **API Layer** | Next.js 15 API Routes (App Router `route.ts` handlers). Auth via anonymous signed-cookie sessions (iron-session or jose). Zod for request/response validation. Deployed on Vercel. |
| **Upload Worker** | Node.js BullMQ worker (`apps/worker-upload/src/`). sharp for thumbnail generation. Writes display thumbnails to public bucket. |
| **Book Placement Engine** | TypeScript module (`packages/templates/src/`). Simple aspect-ratio matcher — stateless, accepts ordered photo list + slot definitions, returns placement JSON (< 500 ms). Hungarian algorithm deferred to v2. |
| **PDF Renderer** | Node.js BullMQ worker (`apps/worker-render/src/`). Playwright (Node bindings) + headless Chromium. Reads placement JSON + image URLs, renders each spread as HTML, exports to PDF via `page.pdf()`. pdf-lib for assembly + font embedding. |
| **Object Storage** | S3-compatible (MinIO in dev; AWS S3 or Cloudflare R2 in prod). Raw uploads in private bucket. CDN-served display images in public bucket. Generated PDFs in ephemeral bucket. |
| **Job Queue** | BullMQ (Redis-backed) for upload and render jobs. Priority queuing. |
| **Database** | PostgreSQL via Prisma ORM (`packages/db/src/`). Stores sessions, photo records, placement JSON, export metadata. |
| **CDN** | Cloudflare. Image resizing at edge (Workers + Images API) for display thumbnails. Next.js Image component for optimised delivery. |

### 8.3 Data Flow Summary

1. User uploads photos → chunked to S3 raw bucket via presigned multipart URLs → upload-complete event enqueued.
2. Upload worker consumes event → generates thumbnails via sharp → writes thumbnail keys to DB → emits processing-complete event.
3. User reviews photos in thumbnail strip and reorders via drag-and-drop (no AI analysis in v1).
4. User confirms → placement engine runs aspect-ratio matching (< 500 ms) → placement JSON stored in DB.
5. Renderer frontend fetches placement JSON → assembles HTML renderer → user sees interactive book.
6. User clicks Download → render job enqueued → Playwright worker produces PDF → signed URL returned to client.

## 13. Out of Scope (V1.0)

- Mobile native apps (iOS / Android). Web is responsive but not native.
- Video or GIF support in books.
- AI-generated captions (user must type captions manually if desired).
- Collaborative book editing (multiple users).
- Custom colour theme overrides or font changes by the user.
- Print-on-demand fulfilment or physical shipping.
- Multiple book sizes (A5, square, US Letter) — V1.0 is A4 portrait only.
- CMYK colour space conversion for professional offset printing.
- Internationalisation (English only in V1.0).

## 14. Appendix

### 14.1 Glossary

| Term | Definition |
|---|---|
| **Spread** | A two-page unit (left + right page) treated as one design canvas. |
| **Slot** | A defined photo placement zone within a spread template. |
| **Affinity Score** | A 0–1 score representing how well a photo matches a slot's visual and semantic requirements. |
| **pHash** | Perceptual hash — a compact fingerprint of image visual content used for near-duplicate detection. |
| **Hungarian Algorithm** | A combinatorial optimisation algorithm that solves the assignment problem in polynomial time. |
| **DPI** | Dots per inch — print resolution measure. 300 DPI is standard for quality photo printing. |
| **Bleed** | Extra image area beyond the page trim edge, ensuring no white borders after cutting. |

### 14.2 Design System Reference

The MemoryBook vintage-retro design system is maintained in a separate Figma file. This PRD does not reproduce the full token catalogue; instead, Section 6.1 documents the subset of tokens relevant to book template construction. Engineering should treat the Figma source as the authoritative reference for all colour, typography, and spacing values.
